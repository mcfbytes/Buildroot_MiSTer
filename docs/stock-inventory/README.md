# Stock image inventory (P0.3)

Full inventory of the shipped stock rootfs (`release_20250402.7z`'s
`files/linux/linux.img`, a 375 MiB ext4 image running Buildroot
2021.02.4 / glibc 2.31 / Linux 5.15.1-MiSTer), plus the kernel config and
device tree extracted from `zImage_dtb`. Every file below is generated by a
script in `scripts/inventory/` (see that directory's `README.md`) so it can
be re-run against a future Buildroot-built image for parity checking
(P2.2's SONAME checker, P2.3's init parity, P3.3's module/firmware infra).

This feeds P0.5 (ABI contract), P0.7 (package mapping), P2.3 (init
parity), and P3.3 (module/firmware infra).

## Headline findings (full detail in each doc)

- **Firmware count corrected: 66, not 72.** PLAN.md §3/§4.1, TASKS.md A5,
  and the verification doc all say stock ships "72 firmware files."
  The actual count of **regular files** under `/usr/lib/firmware` is
  **66**; `find /usr/lib/firmware | wc -l` (no `-mindepth 1`) returns 72
  because it also counts the firmware directory itself (+1) and its 5
  vendor subdirectories as if they were files. Confirmed independently via
  `find -type f` on the extracted tree and cross-checked with
  `debugfs -R "ls -l /usr/lib/firmware"` directly against the raw ext4
  image. **This doc's 66 is the authoritative count going forward** — see
  `firmware.md`. `xow_dongle.bin` (xone/xow dongle firmware) is present in
  stock's firmware set — see `firmware.md`'s redistribution callout for
  P3.2.
- **`/etc/resolv.conf` is a symlink, not a regular file** — one of A8's six
  user-file-restore destinations. `/etc/resolv.conf -> ../tmp/resolv.conf`
  (`/tmp` is tmpfs). Traced through `Downloader_MiSTer`'s actual restore
  code: the plain `open(target, 'wb')` copy follows the symlink, so
  against an *offline-mounted, unbooted* new image the restored content
  lands at `/tmp/resolv.conf` **inside the persisted image**, which the
  `tmpfs` mount over `/tmp` then shadows on first boot — the "restore
  custom resolv.conf" step is a no-op in stock, not a working regular-file
  copy. The other five destinations are genuine regular files. See
  `etc-configs.md`.
- **`/etc/init.d/S45bluetooth` is a symlink to `/bin/bluetoothd`**, not an
  inline script like every other `S`-script. `/bin/bluetoothd` is itself a
  conventional start/stop control script and is stock's actual Bluetooth
  pairing-persistence mechanism: it loop-mounts a small dedicated ext4
  image (`/media/fat/linux/bluetooth`) onto `/var/lib/bluetooth`. Directly
  relevant to P2.4's writable-paths audit and P3.5's Bluetooth parity task.
  See `etc-configs.md`.
- **Default-credential posture**: `root` is the only account with a
  login-capable password hash (every service account is locked); combined
  with `sshd_config`'s `PermitRootLogin yes`, stock ships remote root login
  gated only by a fixed default password. Reported factually (account
  structure, not the hash value) per P3.7/P4.8's needs — see
  `etc-configs.md`. SSH host keys are also baked into the image (same keys
  across every un-regenerated stock install) — same FAQ treatment applies.
- **3 genuine dangling `NEEDED` dependencies** (required but not provided
  anywhere in the image, by SONAME or by filename): `archivemount` needs
  both `libarchive.so.13` and `libfuse.so.2` (neither present);
  `libjack.so.0.0.28` needs `libdb-5.3.so` (not present). These binaries
  cannot actually load as shipped. See `binaries-needed.md`.
- **Kernel config regenerates byte-identical**; **DTS regenerates
  content-identical** (every node/property/value matches; the only textual
  difference is `dtc`-version-dependent cosmetic rendering of one line —
  see `kernel-config-dts.md` for the exact diff and why it's not a real
  difference).
- 507 true ABI shared libraries (SONAME-bearing, excluding ~27 bash
  loadable-builtins that technically carry a self-referential SONAME but
  are never `DT_NEEDED`-resolved); 251 distinct SONAMEs in the union of all
  `DT_NEEDED` across the image (the set P0.7 must map to Buildroot
  packages); 536 ELF binaries; 52 `.ko.xz` kernel modules exactly matching
  A5; BusyBox 1.33.1 with 274 applets.

## Regenerating everything

```sh
scripts/inventory/run-all.sh \
  work/extracted/files/linux/linux.img \
  work/extracted/files/linux/zImage_dtb \
  work/extracted/files/MiSTer
```

See `scripts/inventory/README.md` for the full per-script breakdown, what
each requires, and the determinism guarantees these files rely on to stay
diffable across a future Buildroot-built image.
