## Why `stock.dts` is only "content-identical", not byte-identical

The committed `stock.dts` was decompiled by an earlier/different `dtc`
build than the one installed in this environment. The two texts differ in
exactly one line -- the `/memreserve/` header (`/memreserve/ 0 0x1000;` vs
`/memreserve/\t0x0000000000000000 0x0000000000001000;`, a tab-vs-space and
decimal-vs-hex-zero rendering choice, same value) -- plus whether a
`// version: N` / `// last_comp_version: N` / `// boot_cpuid_phys: N`
banner is emitted and whether nodes are indented with tabs or 4 spaces.
Every node, property, and value in the tree is identical; see
`scripts/inventory/normalize-dts.py` for the exact, narrow normalization
(whitespace collapse + hex/decimal literal equivalence) used to prove this,
and re-run `gen-kernel-config-dts.sh` yourself to reproduce the diff.
