### Resolving the "72 firmware files" figure in PLAN.md §3/§4.1, TASKS.md A5, and the verification doc

Those all say stock ships "72 firmware files". The actual count of
**regular files** is **66**. Reproducing the likely source of
the "72": `find /usr/lib/firmware | wc -l` (i.e. *without* `-mindepth 1`)
counts the firmware directory itself as one line, plus one line per entry
under it -- 66 files + 5 subdirectories + 1 (the dir itself) = **72**, matching the
documented figure exactly. So the existing docs are counting directories
(and the top-level dir itself) as if they were firmware files. **This
doc's 66 is the corrected, authoritative count** (files only,
via `find -type f`, cross-checked against `debugfs -R "ls -l ..."` on the
raw ext4 image directly, not just the extracted tree).

### Redistribution-relevant callout: `xow_dongle.bin`

Present at `/usr/lib/firmware/xow_dongle.bin` in the **stock** image --
this is the Xbox One wireless dongle firmware used by the `xone`/`xow`
driver stack. Directly relevant to **P3.2**'s redistribution question:
stock **does** ship this firmware file today (bundled directly in
`linux.img`, not fetched on demand), so whatever redistribution
constraints apply, MiSTer's existing distribution already accepts them for
this exact file -- P3.2 should confirm the same terms apply to however we
choose to source/ship it (bundle vs on-device fetch) rather than assume a
stricter default.
