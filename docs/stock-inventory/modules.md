### `modules.alias` and the udev autoload mechanism (P3.3)

`modules.alias` non-comment lines: **915**. Mechanism: `depmod` (run
at image build time) generates `modules.alias` from every module's
`MODULE_DEVICE_TABLE` info (`alias <bus-match-pattern> <module-name>`).
**eudev** (stock's hotplug daemon, `S10udev`) matches each hotplugged
device's kernel-generated `MODALIAS` uevent variable against this table
and invokes `modprobe`, which resolves the match through
`modules.alias`/`modules.dep` to load the right `.ko.xz` (and its
dependencies, in the order `modules.dep` specifies) -- no static
device-to-module list is maintained anywhere; it's entirely table-driven
from what's actually plugged in. P3.3 must reproduce this exact chain
(`depmod` at image build + eudev + `modprobe`), not a custom udev-rules
hardcoded module list.
